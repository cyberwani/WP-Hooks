    Add a new option.

    add_option(string $option, mixed $value, mixed $deprecated, bool $autoload) : bool

    You do not need to serialize values. If the value needs to be serialized, then it will be serialized before it is inserted into the database. Remember, resources can not be serialized or added as an option.

    You can create options without values and then update the values later. Existing options will not be updated and checks are performed to ensure that you aren't adding a protected WordPress option. Care should be taken to not name options the same as the ones which are protected.

    Retrieve option value based on name of option.

    get_option(string $option, mixed $default) : mixed

    If the option does not exist or does not have a value, then the return value will be false. This is useful to check whether you need to install an option and is commonly used during installation of plugin options and to test whether upgrading is required.

    If the option was serialized then it will be unserialized when it is returned.

    Update the value of an option that was already added.

    update_option(string $option, mixed $newvalue) : bool

    You do not need to serialize values. If the value needs to be serialized, then it will be serialized before it is inserted into the database. Remember, resources can not be serialized or added as an option.

    If the option does not exist, then the option will be added with the option value, but you will not be able to set whether it is autoloaded. If you want to set whether an option is autoloaded, then you need to use the add_option().
